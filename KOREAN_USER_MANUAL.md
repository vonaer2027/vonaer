## 1. 플랫폼 소개

### 1.1 VONAER 서비스란

VONAER(본에어)는 프라이빗 항공 서비스를 제공하는 온라인 플랫폼입니다. 일반 고객은 웹사이트를 통해 프라이빗 제트, 헬리콥터, 요트 등 럭셔리 모빌리티 서비스를 검색하고 예약 요청을 할 수 있습니다. 관리자는 별도의 관리 페이지를 통해 항공편 정보를 관리하고, 고객 문의를 처리하며, MMS 마케팅 메시지를 발송할 수 있습니다.

### 1.2 Empty Leg 특가 서비스

Empty Leg는 프라이빗 제트의 빈 구간을 뜻합니다. 예를 들어, 한 고객이 서울에서 도쿄로 프라이빗 제트를 이용한 후 비행기가 다시 서울로 돌아와야 할 때, 이 돌아오는 구간은 승객 없이 비행합니다. 이러한 빈 구간을 저렴한 가격에 판매하는 것이 Empty Leg 서비스입니다.

**Empty Leg의 특징**:
- 일반 전세 가격 대비 50-75% 저렴
- 출발 일정과 노선이 미리 정해져 있음
- 선착순 마감 (다른 고객이 먼저 예약하면 이용 불가)
- 일정 변경이 제한적

### 1.3 플랫폼 구성 요소

VONAER 플랫폼은 다음과 같은 요소로 구성되어 있습니다:

| 구성 요소 | 설명 | 접속 주소 |
|-----------|------|-----------|
| 고객 웹사이트 | 일반 고객이 항공편을 검색하고 예약 요청하는 페이지 | https://vonaer.com |
| 관리자 대시보드 | 관리자가 항공편, 예약, MMS를 관리하는 페이지 | https://vonaer.com/admin |
| 자동 수집 시스템 | 6시간마다 자동으로 Empty Leg 항공편을 수집하는 시스템 | GitHub Actions |

### 1.4 지원 언어

웹사이트는 4개 언어를 지원합니다:
- 한국어 (기본)
- 영어 (English)
- 일본어 (日本語)
- 중국어 간체 (中文)

## 2. 시작하기 전에

### 2.1 필요한 준비물

플랫폼을 원활하게 사용하기 위해 다음이 필요합니다:

**관리자 작업용**:
- 인터넷에 연결된 컴퓨터 또는 태블릿
- Chrome, Safari, Edge 등 최신 웹 브라우저
- 관리자 계정 정보 (아이디와 비밀번호)

**MMS 발송용**:
- BizGO(비즈고) MMS 서비스 계정
- 등록된 발신 번호 (1600-9064)
- 충분한 MMS 발송 잔여 건수

### 2.2 관리자 계정 정보 보안

관리자 계정은 시스템의 모든 데이터에 접근할 수 있으므로 다음 보안 수칙을 반드시 지켜주세요:

1. 비밀번호를 다른 사람과 공유하지 마세요
2. 공용 컴퓨터에서 로그인한 후에는 반드시 로그아웃하세요
3. 브라우저에 비밀번호를 저장하지 않는 것을 권장합니다
4. 로그인 상태로 자리를 오래 비우지 마세요

### 2.3 웹사이트 접속 확인

작업을 시작하기 전에 웹사이트가 정상적으로 작동하는지 확인하세요:

**확인 단계**:
1. 브라우저를 열고 https://vonaer.com 에 접속합니다
2. 메인 페이지가 정상적으로 표시되는지 확인합니다
3. 상단 메뉴가 작동하는지 확인합니다
4. Empty Leg 페이지(https://vonaer.com/empty)에 접속하여 항공편 목록이 표시되는지 확인합니다

문제가 있으면 시스템 관리자에게 문의하세요.

## 3. 고객용 웹사이트 사용법

이 섹션은 고객이 웹사이트를 어떻게 사용하는지 설명합니다. 관리자는 고객 문의에 응대할 때 이 정보를 활용할 수 있습니다.

### 3.1 메인 페이지 둘러보기

**접속 주소**: https://vonaer.com

메인 페이지에는 다음 요소가 있습니다:

**상단 영역**:
- VONAER 로고 (클릭 시 메인 페이지로 이동)
- 언어 선택 버튼 (우측 상단)
- 메뉴 버튼 (우측 상단의 세 줄 아이콘)

**중앙 영역**:
- 배경 동영상 또는 이미지
- VONAER 브랜드 소개
- "항공편 예약" 버튼

**하단 영역**:
- 항공편 검색 양식
- 서비스 소개
- 푸터 (개인정보처리방침, 이용약관 링크)

### 3.2 언어 변경 방법

**단계별 안내**:

1. 화면 우측 상단에서 현재 언어가 표시된 버튼을 찾습니다
   - 예: "한국어", "EN", "日本語", "中文"
2. 버튼을 클릭하면 언어 선택 메뉴가 나타납니다
3. 원하는 언어를 클릭합니다
4. 페이지가 자동으로 새로고침되며 선택한 언어로 변경됩니다

**참고**: 언어를 변경해도 입력한 데이터나 로그인 상태는 유지됩니다.

### 3.3 메뉴 탐색

**메뉴 열기**:
1. 화면 우측 상단의 메뉴 버튼(세 줄 아이콘)을 클릭합니다
2. 전체 화면을 덮는 메뉴가 나타납니다

**메뉴 항목**:
| 메뉴 | 설명 |
|------|------|
| Empty Leg | 특가 항공편 목록 페이지로 이동 |
| 제트 | 프라이빗 제트 서비스 소개 |
| 헬리콥터 | 헬리콥터 전세 서비스 소개 |
| eVTOL | 전기 수직 이착륙기 서비스 소개 |
| 요트 | 요트 전세 서비스 소개 |
| 슈퍼요트 | 대형 럭셔리 요트 서비스 소개 |
| 슈퍼카 | 럭셔리 차량 렌탈 서비스 소개 |
| 멤버십 | 회원 프로그램 안내 |
| 회사 소개 | VONAER 회사 정보 |
| 연락처 | 문의처 정보 |

**메뉴 닫기**:
- 화면 우측 상단의 X 버튼을 클릭합니다
- 또는 원하는 메뉴 항목을 클릭하면 해당 페이지로 이동하며 메뉴가 닫힙니다

### 3.4 항공편 검색 및 예약 요청

**검색 양식 작성**:

1. **여행 유형 선택**
   - "편도": 출발지에서 도착지까지만 이동
   - "왕복": 출발지에서 도착지까지 갔다가 다시 돌아옴

2. **출발지 입력**
   - 도시명을 입력합니다 (예: 서울, 인천, 김포)
   - 입력을 시작하면 자동완성 목록이 나타납니다
   - 목록에서 원하는 도시를 선택합니다

3. **도착지 입력**
   - 출발지와 동일한 방식으로 입력합니다

4. **출발일 선택**
   - 달력 아이콘을 클릭합니다
   - 달력에서 원하는 날짜를 선택합니다
   - 오늘 이전 날짜는 선택할 수 없습니다

5. **귀국일 선택** (왕복 선택 시만 표시)
   - 출발일 이후의 날짜를 선택합니다

6. **탑승 인원 설정**
   - 기본값은 1명입니다
   - "+" 버튼으로 인원을 추가합니다
   - "-" 버튼으로 인원을 감소합니다

7. **"검색" 버튼 클릭**

**예약 요청 제출**:

검색 버튼을 클릭하면 예약 요청 창이 나타납니다:

1. **성함 입력**
   - 예약자의 이름을 입력합니다

2. **연락처 입력**
   - 휴대폰 번호를 입력합니다
   - 형식: 010-1234-5678 또는 01012345678

3. **이메일 입력** (선택사항)
   - 이메일 주소를 입력합니다

4. **개인정보 수집 동의**
   - 체크박스를 반드시 선택해야 합니다
   - 동의하지 않으면 예약 요청을 제출할 수 없습니다

5. **"예약 요청 제출" 버튼 클릭**

**예약 요청 완료 후**:
- 성공 메시지가 표시됩니다
- 관리자에게 자동으로 알림이 전송됩니다
- 관리자가 입력된 연락처로 연락을 드립니다

### 3.5 Empty Leg 특가 항공편 이용

**Empty Leg 페이지 접속**:
- 주소: https://vonaer.com/empty
- 또는 메뉴에서 "Empty Leg" 클릭

**항공편 목록 확인**:

각 항공편 카드에는 다음 정보가 표시됩니다:
- 항공편 사진
- 출발지와 도착지 (예: 서울 -> 상하이)
- 출발 날짜
- 항공기 종류 (예: Gulfstream G650)
- 탑승 가능 인원
- 가격 (기체 전체 전세 요금)

**정렬 기능**:
- "가격 낮은 순": 저렴한 항공편부터 표시
- "가격 높은 순": 비싼 항공편부터 표시
- "날짜 빠른 순": 가장 빨리 출발하는 항공편부터 표시 (기본값)
- "날짜 늦은 순": 나중에 출발하는 항공편부터 표시

**예약 방법**:
1. 관심 있는 항공편 카드를 찾습니다
2. "예약 요청" 버튼을 클릭합니다
3. 고객 정보를 입력합니다
4. "예약 요청 제출" 버튼을 클릭합니다

## 4. 관리자 대시보드 완전 가이드

### 4.1 관리자 로그인

**로그인 페이지 접속**:

1. 브라우저를 열고 https://vonaer.com/admin/login 에 접속합니다
2. 로그인 페이지가 표시됩니다

**로그인 과정**:

1. **사용자명 입력**
   - 관리자 아이디를 입력합니다

2. **비밀번호 입력**
   - 비밀번호를 입력합니다
   - 입력 내용은 점(...)으로 표시되어 다른 사람이 볼 수 없습니다

3. **"로그인" 버튼 클릭**

4. **로그인 성공**
   - 관리자 대시보드로 자동 이동합니다

**로그인 실패 시**:
- 아이디 또는 비밀번호가 틀렸다는 메시지가 표시됩니다
- 입력 내용을 확인하고 다시 시도합니다
- 계속 실패하면 시스템 관리자에게 문의합니다

### 4.2 대시보드 화면 구성

로그인 후 표시되는 관리자 대시보드 화면은 다음과 같이 구성되어 있습니다:

**상단 영역**:
- VONAER 로고
- 메뉴 버튼
- "Logout" 버튼 (로그아웃)

**통계 카드 (4개)**:
1. **전체 항공편**: 등록된 총 항공편 수
2. **한국 노선**: 한국 출발/도착 항공편 수
3. **평균 가격**: 모든 항공편의 평균 가격
4. **현재 마진**: 설정된 가격 마진 비율

**탭 메뉴** (5개 탭):
1. **Flights**: 항공편 관리
2. **Bookings**: 예약 요청 관리
3. **MMS Messaging**: MMS 문자 발송
4. **Users**: 고객 정보 관리
5. **Settings**: 가격 마진 설정

### 4.3 로그아웃

**로그아웃 방법**:
1. 화면 우측 상단의 "Logout" 버튼을 클릭합니다
2. 로그인 페이지로 자동 이동합니다

**중요**: 작업을 마친 후에는 반드시 로그아웃하세요. 특히 공용 컴퓨터를 사용할 경우 보안을 위해 로그아웃이 필수입니다.

### 4.4 대시보드 탭 전환

**탭 전환 방법**:
1. 원하는 탭 이름을 클릭합니다
2. 해당 탭의 내용이 화면에 표시됩니다

**각 탭의 기능**:
- **Flights**: 항공편 목록 보기, 추가, 수정, 삭제
- **Bookings**: 고객 예약 요청 확인 및 처리
- **MMS Messaging**: 고객에게 MMS 메시지 발송
- **Users**: 고객 정보 관리 (추가, 수정, 삭제)
- **Settings**: 전체 항공편 가격 마진 설정

## 5. 항공편 관리 상세 가이드

### 5.1 항공편 목록 보기

**접근 방법**:
1. 관리자 대시보드에 로그인합니다
2. "Flights" 탭을 클릭합니다

**목록 화면 구성**:

각 항공편은 카드 형태로 표시되며, 다음 정보를 포함합니다:
- 항공편 이미지
- 출발지 -> 도착지
- 출발 날짜
- 항공기 종류
- 탑승 인원
- 가격 (원가 및 판매가)
- 출처 (FlyXO 또는 JetBay - 자동 수집된 경우)

### 5.2 새 항공편 추가

**추가 과정**:

1. **"+ 새 항공편" 버튼 클릭**
   - Flights 탭 상단의 버튼을 클릭합니다
   - 항공편 입력 양식이 나타납니다

2. **기본 정보 입력**

   **출발지 정보**:
   - 출발 도시: 도시명 입력 (예: Seoul, Tokyo, Shanghai)
   - 출발 국가: 국가 선택 또는 입력 (예: South Korea, Japan, China)

   **도착지 정보**:
   - 도착 도시: 도시명 입력
   - 도착 국가: 국가 선택 또는 입력

   **일정 정보**:
   - 출발 날짜: 달력에서 선택
   - 항공기 종류: 기종명 입력 (예: Gulfstream G650, Challenger 605)

3. **가격 정보 입력**

   **기본 가격**:
   - 원가를 미국 달러($) 단위로 입력합니다
   - 숫자만 입력합니다 (예: 15000)

   **가격 유형**:
   - "고정 가격": 정해진 가격으로 표시
   - "문의": 가격 대신 "Enquire for Price" 표시

   **맞춤 가격** (선택사항):
   - 이 항공편에만 적용되는 특별 가격
   - 입력하면 마진 설정보다 우선 적용됩니다

4. **탑승 정보**
   - 탑승 인원: 최대 탑승 가능 인원수 입력

5. **이미지 업로드**
   - "이미지 업로드" 버튼을 클릭합니다
   - 컴퓨터에서 이미지 파일을 선택합니다
   - 권장 크기: 가로 1200픽셀 이상
   - 지원 형식: JPG, PNG, WebP
   - 여러 장 업로드 가능

6. **"저장" 버튼 클릭**
   - 모든 정보를 확인한 후 저장합니다
   - 성공 메시지가 표시됩니다
   - 항공편이 목록에 추가됩니다

### 5.3 항공편 수정

**수정 과정**:

1. 수정할 항공편 카드를 찾습니다
2. 카드에 있는 "수정" 버튼을 클릭합니다
3. 기존 정보가 입력된 양식이 나타납니다
4. 변경할 정보를 수정합니다
5. "저장" 버튼을 클릭합니다

**수정 가능한 항목**:
- 모든 항공편 정보 (출발지, 도착지, 날짜, 가격 등)
- 이미지 추가 또는 삭제
- 항공기 종류
- 탑승 인원

**참고사항**:
- 수정 즉시 고객 웹사이트에 반영됩니다
- 수정 시간이 자동으로 기록됩니다

### 5.4 항공편 삭제

**삭제 과정**:

1. 삭제할 항공편 카드를 찾습니다
2. 카드에 있는 "삭제" 버튼을 클릭합니다
3. 확인 대화상자가 나타납니다
4. "확인"을 클릭하면 삭제됩니다

**삭제 후 상태**:
- 항공편이 목록에서 즉시 제거됩니다
- 고객 웹사이트에서도 즉시 사라집니다
- 데이터베이스에서는 "비활성" 상태로 보관됩니다

**복구 방법**:
- 일반적인 방법으로는 복구할 수 없습니다
- 복구가 필요한 경우 시스템 관리자에게 문의하세요

### 5.5 항공편 검색 및 필터링

**검색 방법**:

**필터링 방법**:
- "한국 노선만" 필터: 한국 출발 또는 도착 항공편만 표시
- "활성 항공편만" 필터: 현재 예약 가능한 항공편만 표시

## 6. MMS 문자 발송 완전 가이드

### 6.1 MMS 발송 시스템 개요

MMS 발송 기능은 Empty Leg 특가 항공편 정보를 고객에게 문자 메시지로 안내하는 기능입니다. 이 기능은 BizGO(비즈고) SMS/MMS 플랫폼과 연동되어 있습니다.

**발송 시스템 구성**:
- 발신 번호: 1600-9064 (사전 등록된 번호)
- 메시지 형식: MMS (이미지 포함 가능)
- 수신거부 번호: 080-877-6077 (자동 포함)

### 6.2 MMS 발송 전 준비사항

**확인 사항**:

1. **BizGO 잔여 건수 확인**
   - MMS 발송에는 건당 비용이 발생합니다
   - 발송 전 잔여 건수가 충분한지 확인하세요
   - 부족할 경우 BizGO 관리 페이지에서 충전하세요

2. **발송할 항공편 정보 확인**
   - 가격, 날짜, 노선 정보가 정확한지 확인하세요
   - 과거 날짜의 항공편이 아닌지 확인하세요

3. **수신자 목록 확인**
   - 발송할 고객 목록이 최신 상태인지 확인하세요
   - 활성 고객만 발송 대상에 포함됩니다

### 6.3 MMS 발송 단계별 가이드

**1단계: MMS 발송 탭 이동**

1. 관리자 대시보드에 로그인합니다
2. "MMS Messaging" 탭을 클릭합니다
3. MMS 발송 화면이 표시됩니다

**2단계: 항공편 선택**

1. "항공편 선택" 섹션을 찾습니다
2. 드롭다운 메뉴를 클릭합니다
3. 발송할 항공편을 선택합니다
   - 각 항공편은 "노선 / 항공기 / 인원" 형식으로 표시됩니다
   - 예: "서울 -> 상하이 / Gulfstream G650 / 14명"
4. 선택한 항공편의 상세 정보가 카드로 표시됩니다
   - 노선 정보
   - 날짜
   - 좌석 수
   - 가격

**3단계: 메시지 템플릿 선택**

1. "메시지 템플릿 선택" 섹션을 찾습니다
2. 3가지 템플릿 중 하나를 선택합니다:

   **템플릿 1: Empty Leg 특가 안내 (단일 노선)**
   - 편도 항공편용 기본 템플릿
   - 가장 자주 사용되는 형식

   **템플릿 2: Empty Leg 특가 안내 (복수 노선)**
   - 항공기 설명이 더 상세한 템플릿
   - 프리미엄 항공기 홍보 시 적합

   **템플릿 3: Empty Leg 왕복 패키지**
   - 왕복 노선용 템플릿
   - 가는 편과 오는 편 정보를 모두 포함

3. 원하는 템플릿 버튼을 클릭합니다
4. 선택한 템플릿이 강조 표시됩니다

**4단계: 항공편 세부 정보 설정**

템플릿을 선택하면 "항공편 세부 정보 설정" 섹션이 나타납니다.

**편도 항공편 정보 설정**:

| 항목 | 설명 | 예시 |
|------|------|------|
| 출발 시간 | 항공편 출발 시각 | 09:00, 15:30 |
| 도착 시간 | 항공편 도착 시각 | 11:30, 17:45 |
| 비행 시간 | 총 비행 소요 시간 | 1h30m, 2h15m |
| 항공기 설명 | 항공기 종류와 특성 | "비즈니스 최적 기종 - Challenger 605" |
| 항공기 특징 | 항공기의 주요 특징 | "중대형 비즈니스 제트 대표 기종" |

**왕복 패키지 템플릿 추가 설정** (템플릿 3 선택 시):

| 항목 | 설명 |
|------|------|

| 복귀편 도착 시간 | 돌아오는 편 도착 시각 |
| 복귀편 비행 시간 | 돌아오는 편 소요 시간 |
| 복귀편 항공기 설명 | 돌아오는 편 항공기 정보 |
| 복귀편 항공기 특징 | 돌아오는 편 항공기 특징 |

**시간 선택 옵션**:
- 09:00, 09:30, 10:00, 10:30, 11:00, 11:30
- 12:00, 12:30, 13:00, 13:30, 14:00, 14:30
- 15:00, 15:30, 16:00, 16:30, 17:00, 17:30
- 18:00, 18:30, 19:00, 19:30, 20:00
- "원하는 시간대" (시간 미정 시)

**비행 시간 선택 옵션**:
- 1h09m, 1h30m, 1h45m, 2h00m, 2h15m, 2h30m
- 2h45m, 3h00m, 3h30m, 4h00m
- "비행시간" (시간 미정 시)

**항공기 설명 선택 옵션**:
- 비즈니스 최적 기종 - Challenger 605
- 세계 최상위 기종 - Global 7500

- 고급 비즈니스 제트

**5단계: 메시지 내용 확인 및 수정**

1. "메시지 내용" 입력란을 확인합니다
2. 자동 생성된 메시지가 표시됩니다
3. 필요시 내용을 직접 수정할 수 있습니다
4. 수정 후에도 발송 가능합니다

2. 미리보기 창이 나타납니다
3. 다음 내용을 확인합니다:
   - 메시지 전문

   - 템플릿 이름
4. 내용이 올바르면 창을 닫습니다
5. 수정이 필요하면 창을 닫고 내용을 수정합니다

**7단계: 수신자 선택**

1. "수신자 선택" 섹션을 찾습니다
2. 등록된 고객 목록이 표 형태로 표시됩니다
3. 발송할 고객을 선택합니다:
   - 개별 선택: 각 고객 옆의 체크박스 클릭
   - 전체 선택: "전체 선택" 체크박스 클릭
4. 선택된 인원 수가 표시됩니다 (예: "5/20명 선택됨")

**고객 정보 표시 항목**:
- 이름
- 전화번호
- 가입일

**8단계: MMS 발송**

3. 발송 확인 대화상자가 나타납니다
4. "확인"을 클릭합니다
5. 발송 진행 상태가 표시됩니다
6. 발송 완료 메시지가 나타납니다

**발송 결과 확인**:
- 성공: "X명에게 MMS를 성공적으로 발송했습니다"
- 실패: 빨간색 오류 메시지와 원인 표시

### 6.4 MMS 발송 문제 해결

**"발송 실패" 오류 발생 시**:

| 오류 원인 | 해결 방법 |
|-----------|-----------|
| BizGO 잔액 부족 | BizGO 관리 페이지에서 충전 |
| 잘못된 전화번호 | 고객 정보에서 전화번호 확인 및 수정 |
| 발신 번호 미등록 | 시스템 관리자에게 문의 |
| API 연결 오류 | 잠시 후 다시 시도 |
| 메시지 내용 오류 | 메시지에 특수문자 확인 |

**발송 후 확인 방법**:
- 본인 휴대폰 번호로 테스트 발송 권장
- 발송 후 실제 수신 여부 확인
- BizGO 관리 페이지에서 발송 내역 확인

### 6.5 MMS 발송 비용

**비용 안내**:
- MMS 발송은 건당 비용이 발생합니다
- 이미지 포함 MMS: 약 30-50원/건 (요금제에 따라 상이)
- 이미지 없는 SMS: 약 15-25원/건

**비용 절감 팁**:
- 발송 전 수신자 목록을 정리하여 불필요한 발송 방지
- 비활성 고객을 정기적으로 정리
- 테스트 발송은 최소 인원으로 진행

## 7. 설정 변경 가이드

### 7.1 가격 마진 설정

**마진이란**:
마진은 항공편 원가에 추가되는 수수료 비율입니다. 예를 들어, 원가가 $10,000이고 마진이 15%이면 판매가는 $11,500이 됩니다.

**마진 설정 방법**:

1. 관리자 대시보드에 로그인합니다
2. "Settings" 탭을 클릭합니다
3. "가격 마진 설정" 섹션을 찾습니다
4. 현재 마진 값이 표시됩니다
5. 새로운 마진 비율을 입력합니다 (숫자만 입력, % 기호 불필요)
6. "마진 적용" 버튼을 클릭합니다
7. 성공 메시지가 표시됩니다

**마진 적용 규칙**:
- 모든 활성 항공편에 자동 적용됩니다
- 개별 항공편에 "맞춤 가격"이 설정된 경우, 해당 항공편은 맞춤 가격이 우선 적용됩니다
- 마진 변경 즉시 웹사이트에 반영됩니다

**마진 계산 예시**:

| 원가 | 마진 비율 | 판매가 |
|------|-----------|--------|
| $10,000 | 10% | $11,000 |
| $10,000 | 15% | $11,500 |
| $10,000 | 20% | $12,000 |
| $15,000 | 15% | $17,250 |
| $20,000 | 20% | $24,000 |

### 7.2 개별 항공편 가격 설정

특정 항공편에 마진과 다른 가격을 적용하려면:

1. "Flights" 탭에서 해당 항공편을 찾습니다
2. "수정" 버튼을 클릭합니다
3. "맞춤 가격" 항목에 원하는 판매가를 입력합니다
4. "저장" 버튼을 클릭합니다

**주의사항**:
- 맞춤 가격은 마진 설정보다 우선 적용됩니다
- 맞춤 가격을 비워두면 마진이 자동 적용됩니다

### 7.3 고객 정보 관리

**고객 목록 보기**:
1. "Users" 탭을 클릭합니다
2. 등록된 모든 고객 목록이 표시됩니다

**새 고객 추가**:
1. "+ 새 사용자" 버튼을 클릭합니다
2. 고객 정보를 입력합니다:
   - 이름 (필수)
   - 전화번호 (필수)
   - 이메일 (선택)
   - 메모 (선택)
3. "저장" 버튼을 클릭합니다

**고객 정보 수정**:
1. 수정할 고객 행에서 "수정" 버튼을 클릭합니다
2. 정보를 수정합니다
3. "저장" 버튼을 클릭합니다

**고객 삭제**:
1. 삭제할 고객 행에서 "삭제" 버튼을 클릭합니다
2. 확인 대화상자에서 "확인"을 클릭합니다

**참고**: 예약 요청을 통해 접수된 고객은 자동으로 등록됩니다.

## 8. 자동 항공편 수집 시스템 (GitHub Actions)

### 8.1 자동 수집 시스템이란

자동 수집 시스템(GitHub Actions)은 외부 Empty Leg 정보 사이트에서 항공편 정보를 자동으로 수집하여 VONAER 데이터베이스에 저장하는 시스템입니다. 사람이 수동으로 항공편을 하나씩 입력할 필요 없이, 프로그램이 자동으로 최신 항공편 정보를 가져옵니다.

**수집 대상 사이트**:
1. **FlyXO**: 미국 기반 글로벌 Empty Leg 정보 플랫폼
2. **JetBay (Jet-Bay)**: 한국 기반 프라이빗 제트 정보 플랫폼

**수집 대상 항공편**:
- 서울(인천/김포) 출발 또는 도착 노선
- 현재 날짜 이후 출발 예정인 항공편

### 8.2 자동 수집 실행 일정

**정기 실행**:
- 매 6시간마다 자동 실행
- 실행 시간: 0시, 6시, 12시, 18시 (UTC 기준)
- 한국 시간으로는 약 9시, 15시, 21시, 3시

**수동 실행**:
- GitHub 웹사이트에서 직접 실행할 수 있습니다
- 긴급하게 최신 정보가 필요할 때 사용합니다

### 8.3 GitHub Actions 모니터링 방법

**GitHub 접속**:
1. 웹 브라우저에서 GitHub (https://github.com) 에 접속합니다
2. VONAER 프로젝트 저장소로 이동합니다
3. 상단 메뉴에서 "Actions" 탭을 클릭합니다

**실행 기록 확인**:
1. "Actions" 탭에서 "Scrape Empty Leg Flights" 워크플로우를 찾습니다
2. 최근 실행 기록 목록이 표시됩니다
3. 각 실행의 상태를 확인합니다:
   - 녹색 체크: 성공
   - 빨간색 X: 실패
   - 주황색 원: 진행 중

**실행 상세 정보 확인**:
1. 특정 실행 기록을 클릭합니다
2. 실행 요약(Summary)이 표시됩니다:
   - 실행 시간
   - 수집된 항공편 수
   - 성공/실패 여부
3. 상세 로그를 보려면 "scrape" 작업을 클릭합니다

### 8.4 수동으로 수집 실행하기

**수동 실행 방법**:

1. GitHub에서 "Actions" 탭으로 이동합니다
2. 좌측 목록에서 "Scrape Empty Leg Flights"를 클릭합니다
3. "Run workflow" 버튼을 클릭합니다
4. 드롭다운 메뉴에서 설정을 확인합니다:
   - Branch: main (기본값)
   - Enable debug mode: false (기본값), 문제 해결 시 true
5. "Run workflow" 버튼을 클릭합니다
6. 워크플로우가 시작됩니다

**실행 완료 확인**:
1. 페이지를 새로고침합니다
2. 새로운 실행 기록이 목록에 나타납니다
3. 실행 상태가 녹색 체크로 변경될 때까지 기다립니다
4. 완료 후 관리자 대시보드에서 새 항공편을 확인합니다

### 8.5 수집 시스템 작동 원리

**수집 과정 (자동)**:

1. **시작**: GitHub Actions가 예약된 시간에 자동 시작
2. **환경 준비**: 필요한 프로그램과 브라우저 설치
3. **FlyXO 수집**: FlyXO 사이트에서 서울 관련 항공편 검색 및 수집
4. **JetBay 수집**: JetBay 사이트에서 한국 관련 항공편 검색 및 수집
5. **데이터 통합**: 두 사이트의 결과를 하나로 합침
6. **중복 제거**: 동일한 항공편이 두 사이트에 있으면 하나만 유지
7. **유효성 검증**:
   - 과거 날짜 항공편 제외
   - 위치 정보가 없는 항공편 제외
   - 가격 정보가 없는 항공편 제외
8. **데이터베이스 저장**: 유효한 항공편을 Supabase 데이터베이스에 저장
9. **완료 보고**: 수집 결과 요약 생성

**수집되는 정보**:
- 출발지 (도시, 국가)
- 도착지 (도시, 국가)
- 출발 날짜
- 항공기 종류
- 탑승 가능 인원
- 가격
- 항공편 이미지
- 출처 (FlyXO 또는 JetBay)

### 8.6 수집 실패 시 대처

**실패 원인 및 해결책**:

| 원인 | 증상 | 해결책 |
|------|------|--------|
| 사이트 변경 | 수집 수가 0 또는 매우 적음 | 시스템 관리자에게 문의 |
| 네트워크 오류 | 실행 중 오류 발생 | 자동으로 재시도됨, 계속 실패 시 관리자 문의 |
| 데이터베이스 오류 | 저장 단계에서 실패 | 시스템 관리자에게 문의 |
| 타임아웃 | 30분 초과로 중단 | 자동으로 재시도됨 |

**자동 재시도**:
- 실패 시 자동으로 최대 3회까지 재시도합니다
- 각 재시도 사이에 30초 대기합니다

**알림 확인**:
- 실패 시 GitHub Actions 페이지에서 빨간색 X 표시로 확인 가능
- 상세 오류 내용은 실행 로그에서 확인

### 8.7 수집된 항공편 관리

**새로 수집된 항공편 확인**:
1. 관리자 대시보드의 "Flights" 탭으로 이동합니다
2. 최근 추가된 항공편을 확인합니다
3. 각 항공편 카드에서 출처(FlyXO/JetBay)를 확인할 수 있습니다

**수집된 항공편 검토**:
- 가격 정보가 정확한지 확인합니다
- 날짜 정보가 정확한지 확인합니다
- 필요 시 정보를 수정합니다
- 부적절한 항공편은 삭제합니다

**권장 관리 방법**:
- 매일 아침 새로 수집된 항공편을 검토합니다
- 판매에 적합한 항공편을 선별합니다
- 필요 시 MMS로 고객에게 안내합니다

## 9. 예약 요청 처리 워크플로우

### 9.1 예약 요청 확인

**알림 경로**:
1. **Google Chat 알림**: 예약 요청 즉시 관리자 Google Chat으로 알림 전송
2. **관리자 대시보드**: "Bookings" 탭에서 모든 예약 요청 확인 가능

**예약 요청 목록 확인**:
1. 관리자 대시보드에 로그인합니다
2. "Bookings" 탭을 클릭합니다
3. 예약 요청 목록이 표시됩니다

**표시 정보**:
- 고객 이름
- 연락처
- 이메일 (입력한 경우)
- 요청한 항공편 정보
- 요청 일시
- 연락 완료 여부

### 9.2 예약 요청 상세 보기

1. 확인할 예약 요청 행에서 "상세보기" 버튼을 클릭합니다
2. 상세 정보 창이 나타납니다:
   - 고객 연락처 정보
   - 요청한 항공편 상세 정보
   - 항공편 이미지
   - 요청 시간

### 9.3 고객 연락 및 처리

**연락 프로세스**:

1. **고객 연락**
   - 예약 요청에 있는 연락처로 전화합니다
   - 고객의 요청 내용을 확인합니다
   - 항공편 이용 가능 여부를 안내합니다

2. **이용 가능 여부 확인**
   - 해당 항공편이 아직 예약 가능한지 확인합니다
   - Empty Leg의 경우 선착순이므로 빠르게 확인해야 합니다

3. **세부 사항 안내**
   - 정확한 출발 시간 안내
   - 공항 정보 안내
   - 가격 및 결제 방법 안내
   - 필요한 서류 안내

4. **예약 확정 또는 취소**
   - 고객이 예약을 확정하면 결제 절차 진행
   - 고객이 취소하면 요청 종료

### 9.4 연락 완료 표시

고객 연락 완료 후:
1. 해당 예약 요청에서 "연락 완료" 버튼을 클릭합니다
2. 상태가 "연락 완료"로 변경됩니다
3. 체크 표시 아이콘이 나타납니다

**중요**: 연락 완료 표시를 하면 미처리 건수에서 제외되어 다른 팀원도 처리 완료 여부를 알 수 있습니다.

### 9.5 예약 요청 삭제

불필요한 예약 요청 삭제:
1. 삭제할 요청에서 "삭제" 버튼을 클릭합니다
2. 확인 대화상자에서 "확인"을 클릭합니다

**삭제 적합한 경우**:
- 스팸 요청
- 중복 요청
- 고객이 취소한 요청
- 처리 완료 후 정리

## 11. 문제 해결 가이드

### 11.1 로그인 문제

**증상: 로그인이 되지 않음**

| 원인 | 해결 방법 |
|------|-----------|
| 아이디/비밀번호 오타 | 대소문자 구분하여 정확히 입력 |
| Caps Lock 켜짐 | Caps Lock 끄고 다시 시도 |
| 브라우저 캐시 문제 | 브라우저 캐시 삭제 후 재시도 |
| 계정 잠김 | 시스템 관리자에게 문의 |

**해결 단계**:
1. 아이디와 비밀번호를 천천히 다시 입력합니다
2. 다른 브라우저로 시도합니다
3. 브라우저의 캐시와 쿠키를 삭제합니다
4. 여전히 안 되면 시스템 관리자에게 문의합니다

### 11.2 데이터 로딩 문제

**증상: 항공편/예약 목록이 표시되지 않음**

| 원인 | 해결 방법 |
|------|-----------|
| 인터넷 연결 끊김 | 인터넷 연결 확인 |
| 서버 일시 오류 | 페이지 새로고침 |
| 세션 만료 | 다시 로그인 |

**해결 단계**:
1. 인터넷 연결을 확인합니다
2. 페이지를 새로고침합니다 (F5 또는 Ctrl+R)
3. 로그아웃 후 다시 로그인합니다
4. 다른 브라우저로 시도합니다

### 11.3 MMS 발송 문제

**증상: MMS 발송 실패**

| 오류 메시지 | 원인 | 해결 방법 |
|-------------|------|-----------|
| "잔액 부족" | BizGO 발송 건수 부족 | BizGO 충전 |
| "잘못된 번호" | 전화번호 형식 오류 | 번호 형식 확인 (010-xxxx-xxxx) |
| "발신번호 미등록" | 발신번호 인증 필요 | 시스템 관리자 문의 |
| "서버 오류" | 일시적 연결 문제 | 잠시 후 재시도 |

**발송 전 확인사항**:
1. 수신자 전화번호가 올바른지 확인
2. 메시지 내용에 오류가 없는지 확인
3. BizGO 잔여 건수가 충분한지 확인

### 11.4 항공편 저장 문제

**증상: 항공편 추가/수정 실패**

| 원인 | 해결 방법 |
|------|-----------|
| 필수 정보 누락 | 모든 필수 항목 입력 |
| 이미지 용량 초과 | 5MB 이하 이미지 사용 |
| 잘못된 날짜 형식 | 달력에서 날짜 선택 |

**해결 단계**:
1. 모든 필수 입력 항목을 확인합니다
2. 이미지 크기를 확인합니다 (5MB 이하 권장)
3. 날짜를 달력에서 다시 선택합니다
4. 저장 버튼을 다시 클릭합니다

### 11.5 자동 수집 문제

**증상: 새 항공편이 수집되지 않음**

| 원인 | 해결 방법 |
|------|-----------|
| GitHub Actions 오류 | GitHub에서 실행 로그 확인 |
| 수집 대상 사이트 변경 | 시스템 관리자에게 문의 |
| 수집 대상 항공편 없음 | 정상 상황, 대기 |

**확인 방법**:
1. GitHub Actions 페이지에서 최근 실행 상태 확인
2. 빨간색 X 표시가 있으면 실패한 것
3. 로그를 확인하여 오류 원인 파악
4. 시스템 관리자에게 문의

## 12. 용어 사전

| 용어 | 설명 |
|------|------|
| **Empty Leg** | 프라이빗 제트가 승객 없이 이동하는 빈 구간. 저렴한 특가로 제공됨 |
| **전세** | 항공기 전체를 빌리는 것. 좌석 단위가 아닌 기체 전체 대여 |
| **원가** | 항공사 또는 브로커가 제시한 기본 가격 |
| **판매가** | 마진이 포함된 최종 고객 가격 |
| **마진** | 원가에 추가되는 수수료 비율 (%) |
| **맞춤 가격** | 개별 항공편에 설정한 특별 가격. 마진보다 우선 적용 |
| **활성/비활성** | 항공편의 표시 여부. 활성은 고객에게 보임, 비활성은 숨김 |
| **크롤링** | 외부 웹사이트에서 자동으로 정보를 수집하는 과정 |
| **GitHub Actions** | 자동화된 작업을 실행하는 시스템. 항공편 자동 수집에 사용 |
| **MMS** | 멀티미디어 메시지. 이미지가 포함된 문자 메시지 |
| **SMS** | 단순 문자 메시지. 이미지 없음 |
| **BizGO** | MMS/SMS 발송 서비스 제공업체 |
| **Supabase** | 데이터베이스 서비스 제공업체. 모든 데이터 저장 |
